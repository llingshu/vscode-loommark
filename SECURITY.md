# Security Policy

## Supported Versions

The latest released minor version receives security fixes. Older versions may be asked to upgrade
before a fix is provided.

## Reporting A Vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub's **Report a vulnerability**
button in the repository Security tab to submit a private advisory.

Include the affected version, reproduction steps, impact, and any suggested mitigation. You should
receive an acknowledgement within seven days. Please allow maintainers time to investigate and
coordinate a release before public disclosure.

## Scope

Reports involving document corruption, unintended network access, Webview content-security-policy
bypasses, command execution, or unsafe file access are in scope. Vulnerabilities in dependencies
should also be reported upstream when appropriate.
